泛型#

函数签名中使用泛型#

尝试编写一个寻找数组中最大值的函数:

fn largest_i32(list: &[i32]) -> &i32 {
    let mut largest = &list[0];

    for item in list.iter() {
        if item > largest {
            largest = item;
        }
    }

    largest
}

该函数仅支持i32类型，如果我们需要一个char类型的largest_char函数，应该会是这样:

fn largest_char(list: &[char]) -> &char {
    let mut largest = &list[0];

    for item in list.iter() {
        if item > largest {
            largest = item;
        }
    }

    largest
}

可以发现，我们在编写一些逻辑重复，仅类型不同的代码，为了能够支持更多的类型而不需要手写它们的largest函数，可以将这部分代码中的类型使用泛型替换。

所以泛型版本应该是这么写的:

fn largest_t<T>(list: &[T]) -> &T {  
    let mut largest = &list[0];  
  
    for item in list.iter() {  
        if item > largest {  
            largest = item;  
        }  
    }  
  
    largest  
}

首先我们在函数名后添加了<T>，这是一个泛型参数，参数list也变成了一个泛型slice，同样的，返回值也进行了修改，而我们的函数体不需要任何修改。

我们看看如何调用这个函数:

fn main() {
    let number_list = vec![34, 50, 25, 100, 65];

    let result = largest_t(&number_list);
    println!("The largest number is {}", result);

    let char_list = vec!['y', 'm', 'a', 'q'];

    let result = largest_t(&char_list);
    println!("The largest char is {}", result);
}

i32和char类型的largest_t方法调用形式是一样的，因为rust可以通过传入的参数类型自行推断泛型T的具体类型，所以我们不需要为largest_t提供具体类型信息。

但是这个largest_t函数目前还是无法运行的: 因为我们还少考虑了一件事，我们的函数体中含有了&T的比较操作，但当前这个泛型参数T代表了任意类型，并不是所有类型都能够支持比较操作符，对于不支持比较的类型，我们应该限制它的传入。 编译器给出了提示，考虑对类型参数T加上限制，给出的方案是使用<T: std::cmp::PartialOrd>，这是一个trait，我们后文会解析，它在这里的作用就是限制泛型参数T的具体类型必须是可比较的。 添加上这个trait后，代码就可以执行了。

除了在函数签名中使用泛型参数来定义泛型函数，定义结构体和枚举时也可以使用泛型参数。

结构体定义中使用泛型#

struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    let integer = Point { x: 5, y: 10 }; // ✅
    let float = Point { x: 1.0, y: 4.0 }; // ✅
    let e = Point { x: 1, y: 4.0 }; // ❌  实例化时x和y的具体类型不一致
}

泛型参数的声明位于结构体名称之后，使用尖括号包裹，在成员定义中可以使用。 在这里，我们仅声明了一个泛型参数T，成员x和成员y都是T类型，所以x和y对应的具体类型也是一致的，例如都是i32或者都是f64。 如果希望x和y可以是不同类型，我们需要声明两个泛型参数:

struct Point<T, U> {
    x: T,
    y: U,
}

fn main() {
    let both_integer = Point { x: 5, y: 10 };
    let both_float = Point { x: 1.0, y: 4.0 };
    let integer_and_float = Point { x: 5, y: 4.0 };
}

枚举定义中使用泛型#

枚举定义使用泛型之前已经见过了:

enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}

方法定义中使用泛型#

方法定义可以在泛型参数上定义，也可以针对具体类型定义

struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}
// 仅Point<f32>拥有
impl Point<f32> {
    fn distance_from_origin(&self) -> f32 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}

fn main() {
    let p = Point { x: 5, y: 10 };

    println!("p.x = {}", p.x());
}

struct Point<T, U> {
    x: T,
    y: U,
}

impl<T, U> Point<T, U> {
	// 在方法的签名中定义另外的泛型参数<V, W>
    fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> {
        Point {
            x: self.x,
            y: other.y,
        }
    }
}

fn main() {
    let p1 = Point { x: 5, y: 10.4 };
    let p2 = Point { x: "Hello", y: 'c' };

    let p3 = p1.mixup(p2);

    println!("p3.x = {}, p3.y = {}", p3.x, p3.y);
}

rust的泛型原理

rust的泛型原理是生成代码式，编译器根据我们调用泛型代码时提供的类型，生成对应的具体类型代码，例如上文中，编译器会为泛型函数largest_t生成具体的i32版本和char版本，这和你手写重复代码没有区别。所以从编译后得到的最终结果来看，使用泛型没有性能上的损失。

trait#

trait在rust中常和泛型搭配使用，它和go语言中的接口(interface)很像，但有一点不同的是，go语言中接口的实现是隐式的，而rust中实现trait需要显式的声明。

如何理解trait和泛型的关系

当我们简单声明一个泛型T，此时它可以代表任意类型，但任意类型并不意味着我们可以对T做任意操作，相反，此时我们无法对T做任何操作，因为我们无法控制外部传入的具体类型一定支持我们所做的操作。所以，我们对T要做的操作越多，相应对T的限制就越多。trait就是用来附加这些限制的手段。 或者换个角度，trait是给泛型T提供行为的手段，一个光秃秃的泛型T，它不具有任何行为，通过trait可以给泛型T添加行为，从而支持我们需要的操作。

定义trait#

// 使用trait关键字，pub关键字允许在其他crate实现该trait
pub trait Summary {
	// 仅给方法签名而不需要函数体，以分号结尾
    fn summarize(&self) -> String;
}

为类型实现trait#

pub struct NewsArticle {
    pub headline: String,
    pub location: String,
    pub author: String,
    pub content: String,
}

// 使用impl+for关键字
impl Summary for NewsArticle {
	// 提供具体函数体
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}

实现trait的作用域问题

你可以在自己的crate内为内部类型实现外部公共的trait，也可以在自己的crate中为外部的类型实现我们内部的trait。 但是你不能在自己的crate中，为外部类型实现外部的trait。这个规则是为了限制两个crate都为同一个类型实现同一个trait，rust编译器无法知道应该使用哪个。

trait的默认实现#

我们可以在定义trait时就为方法签名提供具体实现，这种实现称为默认实现。

pub trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }
}

为类型实现trait时，如果只有该trait仅有默认实现，类型可以不提供具体实现而直接继承该trait的所以默认实现:

pub trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }
}

pub struct NewsArticle {
    pub headline: String,
    pub location: String,
    pub author: String,
    pub content: String,
}

// 具体实现留空即可
impl Summary for NewsArticle {}

如果类型实现trait时为默认实现也提供了具体实现，则会发生重载。

如果trait除了默认实现外还有其他方法定义，则类型实现该trait时，需要为所有非默认实现提供具体实现:

pub trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }

    fn summarize_author(&self) -> String;
}

pub struct NewsArticle {
    pub headline: String,
    pub location: String,
    pub author: String,
    pub content: String,
}

// NewsArticle直接继承了summarize，而summarize_author必须提供具体实现
impl Summary for NewsArticle {
    fn summarize_author(&self) -> String {
        format!("@{}", self.author)
    }
}

因为在定义trait时，默认实现就提供了具体代码，所以在默认实现中，我们可以调用该trait的其他默认实现或方法:

pub trait Summary {
    fn summarize(&self) -> String {
        format!("author is {}, content is {}", self.summarize_author(), self.summarize_content())
    }

    fn summarize_author(&self) -> String;

    fn summarize_content(&self) -> String;
}

pub struct NewsArticle {
    pub headline: String,
    pub location: String,
    pub author: String,
    pub content: String,
}

impl Summary for NewsArticle {
    fn summarize_author(&self) -> String {
        format!("@@@{}", self.author)
    }

    fn summarize_content(&self) -> String {
        format!("!!!{}", self.content)
    }
}

fn main() {
    let article = NewsArticle {
        headline: String::from("headline"),
        location: String::from("location"),
        author: String::from("author"),
        content: String::from("content"),
    };
    println!("{}", article.summarize());
}

默认实现特性使得我们可以快速为类型添加方法，这在其他编程语言中通常需要用继承或组合其他结构体的形式实现。

使用trait#

正如go中的接口一样，我们可以在函数参数中使用trait替换具体类型，这使得该函数不再与某一具体类型耦合，仅与trait所代表的一类行为相关，即——面对抽象而不是具体。

// 使用impl+trait替换具体类型，item参数允许传入所有实现了Summary的类型
pub fn notify(item: impl Summary) {
    println!("Breaking news! {}", item.summarize());
}

函数中的impl+trait形式其实是一种语法糖，我们一般在比较直观简单的场景使用它。而它原本的形式是:

pub fn notify<T: Summary>(item: T) {
    println!("Breaking news! {}", item.summarize());
}

这种写法称为trait bound，trait bound与泛型参数搭配，表达泛型参数的可用行为。使用trait bound的写法可以支持更复杂的场景。

impl+trait和trait bound的区别

这两种写法是等价的，item1和item2可以是不同的类型

pub fn notify(item1: impl Summary, item2: impl Summary)

pub fn notify<T: Summary, U: Summary>(item1:T, item2:U)

如果我们需要限制传入的item1和item2必须是相同的类型，则只能用trait bound语法:

pub fn notify<T: Summary>(item1: T, item2: T)

函数参数可以使用的trait不止一个，我们可以使用+来加入更多trait:

pub fn notify(item: impl Summary + Display)
or
pub fn notify<T: Summary + Display>(item: T)

这表示item参数需要同时实现Summary和Display这两个trait。

rust还提供了where语法优化长串的trait bound:

fn some_function<T: Display + Clone, U: Clone + Debug>(t: T, u: U) -> i32{}

fn some_function<T, U>(t: T, u: U) -> i32
    where T: Display + Clone,
          U: Clone + Debug
{}

泛型结构体实现特定trait bound方法#

use std::fmt::Display;

struct Pair<T> {
    x: T,
    y: T,
}

impl<T> Pair<T> {
    fn new(x: T, y: T) -> Self {
        Self {
            x,
            y,
        }
    }
}

// 仅当T实现了Display + PartialOrd时，该Pait<T>才能调用cmp_display方法
impl<T: Display + PartialOrd> Pair<T> {
    fn cmp_display(&self) {
        if self.x >= self.y {
            println!("The largest member is x = {}", self.x);
        } else {
            println!("The largest member is y = {}", self.y);
        }
    }
}


struct I32 {  
    x: i32,  
}  
  
impl I32 {  
    fn new(x: i32) -> Self {  
        Self {  
            x,  
        }  
    }  
}

fn main() {
	// i32类型实现了Display + PartialOrd
    let pair = Pair::new(2, 3);
    pair.cmp_display();
    
	// f64也可以
    let pair = Pair::new(3.0, 2.0);
    pair.cmp_display();

	// ❌自定义的I32类型调用失败
	let pair = Pair::new(I32::new(2), I32::new(3));  
	pair.cmp_display();
}

Blanket implementations#

对任何满足特定 trait bound 的类型实现 trait 被称为 blanket implementations，他们被广泛的用于 Rust 标准库中。例如，标准库为任何实现了 Display trait 的类型实现了 ToString trait。这个 impl 块看起来像这样：

impl<T: Display> ToString for T {
    // --snip--
}

因为标准库有了这些 blanket implementation，我们可以对任何实现了 Display trait 的类型调用由 ToString 定义的 to_string 方法。例如，可以将整型转换为对应的 String 值，因为整型实现了 Display：

let s = 3.to_string();
这里感觉和默认实现有点类似，都是为实现了trait的类型增加方法。
