Rust 编译#

rust源码首先经过词法分析变成一个个Token，再经过语法分析变为抽象语法树AST，之后rust并没有直接进行机器码的生成，而是将AST降级为高级中间语言HIR，HIR被用于rust编译器对代码进行类型检查、方法查找等工作；接着HIR再次降级为中级中间语言MIR，MIR用于借用检测、代码生成(宏、泛型)，最终MIR转化为LLVM中间语言，再转化为机器码。

词法结构#

关键字-keywords#

严格关键字Strict：

as/ break/ const/ continue/ crate/ if/ else/ struct/ enum/ true/ false/ fn/ for/ in/ let/ loop/ impl/ mod/ match/ move / mut /pub/ ref/ return/ self/ Self/ static/ super/ trait/ type/ unsafe/ use/ where/ while /async/ await/ dyn / main

保留字Reserved：

abstract/ become/ box/ do/ final/ macro/ override/ priv/ typeof/ unsized/ virtual/ yield / try

保留不代表将来一定会使用

弱关键字Weak：

  • 2018 Edition: union, ‘static
  • 2015 Edition: dyn (在2018Editon中已经是严格关键字)

标识符-identifier#

  1. 标识符由字母、数字和下划线组成，且必须以字母或下划线开头。

  2. 禁止使用Rust的保留字（例如match、if、while等等）作为标识符，这会导致编译错误。

  3. _（单个下划线）表示忽略变量的值，可以用于占位符。

注释-comment#

rust注释可以分代码注释、文档注释和模块/包注释

代码注释#

代码注释分行注释和块注释：

// 行注释
fn main() {
    // 我是ivan...
    let name = "ivan";
    let age = 18; // 今年好像是18岁
}

// 块注释
fn main() {
    /*
        我
        是
        i
        v
        a
        n
        ... 哎，好长!
    */
    let name = "ivan";
    let age = "???"; // 今年其实。。。挺大了
}

可见rust的代码注释和其他语言没有什么区别，要介绍的重点是rust的文档注释

文档注释#

当查看一个 crates.io 上的包时，往往需要通过它提供的文档来浏览相关的功能特性、使用方式，这种文档就是通过文档注释实现的。

Rust 提供了 cargo doc 的命令，可以用于把这些文档注释转换成 HTML 网页文件，最终展示给用户浏览，这样用户就知道这个包是做什么的以及该如何使用。

文档行注释，使用三个斜杠///


#![allow(unused)]
fn main() {
/// `add_one` 将指定值加1
///
/// # Examples
///
/// ```
/// let arg = 5;
/// let answer = my_crate::add_one(arg);
///
/// assert_eq!(6, answer);
/// ```
  pub fn add_one(x: i32) -> i32 {
      x + 1
  }
}

以上代码有几点需要注意：

  • 文档注释需要位于 lib 类型的包中，例如 src/lib.rs 中。
  • 文档注释可以使用 markdown语法！例如 # Examples 的标题，以及代码块高亮。
  • 被注释的对象需要使用 pub 对外可见，记住：文档注释是给用户看的，内部实现细节不应该被暴露出去

文档块注释，使用/** … */


#![allow(unused)]
fn main() {
/** `add_two` 将指定值加2

Examples

```
let arg = 5;
let answer = my_crate::add_two(arg);

assert_eq!(7, answer);
```
*/
  pub fn add_two(x: i32) -> i32 {
      x + 2
  }
}

运行 cargo doc 可以直接生成 HTML形式的文档，放入target/doc目录下。

模块/包注释#

除了函数、结构体等 Rust 项的注释，你还可以给包和模块添加注释，需要注意的是，这些注释要添加到包、模块的最上方！

与之前的任何注释一样，包级别的注释也分为两种：行注释 //! 和块注释 /*! ... */。

词条-token#

  1. 语言项 （item）
  2. 块（block）
  3. 语句（Stmt）
  4. 表达式 （Expr）
  5. 模式 （Pattern）
  6. 关键字 （Keyword）
  7. 标识符 （Ident）
  8. 字面量 （Literal）
  9. 生命周期 （Lifetime）
  10. 可见性 （Vis）
  11. 标点符号（Punctuation）
  12. 分隔符（delimiter）
  13. 词条树（Token Tree）
  14. 属性（Attribute）
