    Rust错误处理

    2023-01-16 · 壹次心
    目录

    rust中的错误分为可恢复错误（recoverable）和 不可恢复错误（unrecoverable）。可恢复错误使用Result<T,E>枚举，不可恢复错误使用panic！宏。

    不可恢复错误——panic!宏#

    当出现 panic 时，程序默认会开始 展开（unwinding），这意味着 Rust 会回溯栈并清理它遇到的每一个函数的数据，不过这个回溯并清理的过程有很多工作。另一种选择是直接 终止（abort），这会不清理数据就退出程序。那么程序所使用的内存需要由操作系统来清理。如果你需要项目的最终二进制文件越小越好，panic 时通过在 Cargo.toml 的 [profile] 部分增加 panic = 'abort'，可以由展开切换为终止。例如，如果你想要在release模式中 panic 时直接终止：

    [profile.release]
panic = 'abort'

    panic!宏的使用#

    fn main() {
    panic!("crash and burn");
}

    可恢复错误——Result<T,E>枚举#

    enum Result<T, E> {
    Ok(T),
    Err(E),
}

    T 和 E 是泛型类型参数。通常处理可恢复错误时，函数将会返回一个Result，调用者可使用match进行匹配。

    use std::fs::File;

fn main() {
    let f = File::open("hello.txt");

    let f = match f {
        Ok(file) => file, // 如果成功,返回值的file赋值给f
        Err(error) => {
            panic!("Problem opening the file: {:?}", error)
        },
    };
}

    如果有需要匹配多种错误的场景，可以再嵌套一层match:

    use std::fs::File;
use std::io::ErrorKind;

fn main() {
    let f = File::open("hello.txt");

    let f = match f {
        Ok(file) => file,
        Err(error) => match error.kind() {
	        // 如果是未找到文件，则创建
            ErrorKind::NotFound => match File::create("hello.txt") {
                Ok(fc) => fc,
                Err(e) => panic!("Problem creating the file: {:?}", e),
            },
            other_error => panic!("Problem opening the file: {:?}", other_error),
        },
    };
}

    使用太多match会导致代码冗长，可以使用unwrap和expect进行简写

    use std::fs::File;

fn main() {
    let f = File::open("hello.txt").unwrap();
}

    unwrap内做了match操作，如果出现错误，unwrap_failed方法中会调用panic!宏

    使用expect还可以传入发生panic!时的自定义错误信息，帮助我们留下更多错误信息。

    use std::fs::File;

fn main() {
    let f = File::open("hello.txt").expect("Failed to open hello.txt");
}

    错误传播(propagating)#

    在编写一个可能出错的函数代码时，我们并不一定要自己处理，有时候将错误返回出去是更好的选择，调用者比我们更加清楚如何处理这些错误。

    例如一个从文件中读取用户名的函数。如果文件不存在或不能读取，这个函数会将这些错误返回给调用它的代码：

    use std::io;
use std::io::Read;
use std::fs::File;

fn read_username_from_file() -> Result<String, io::Error> {
    let f = File::open("hello.txt");

    let mut f = match f {
        Ok(file) => file,
        Err(e) => return Err(e), // 将错误返回
    };

    let mut s = String::new();

    match f.read_to_string(&mut s) {
        Ok(_) => Ok(s),
        Err(e) => Err(e), // 将错误返回
    }
}

    这种在成功时返回表达式，在失败时直接return Error的形式比较固定，所以rust提供了一种简写方式，使用?。

    use std::io;
use std::io::Read;
use std::fs::File;

fn read_username_from_file() -> Result<String, io::Error> {
    let mut f = File::open("hello.txt")?;
    let mut s = String::new();
    f.read_to_string(&mut s)?;
    Ok(s)
}

    这段代码和上一段等价，但是省去了编写match代码。

    这段代码甚至还能够被?简化，通过链式使用?。

    use std::io;
use std::io::Read;
use std::fs::File;

fn read_username_from_file() -> Result<String, io::Error> {
    let mut s = String::new();

    File::open("hello.txt")?.read_to_string(&mut s)?;

    Ok(s)
}
